## Internal Tracking 演算法(不可見的核心運算)— Steps 0-3

**以下步驟純屬內部運算,絕對禁止將過程印在最終畫面上。每批馬完成後,直接輸出其對應的 [第二部分] 結果。**

### 📅 賽事日期鎖定 (Race Date Lock) [最優先執行]
從用戶提供的原始數據中識別並鎖定今日賽事日期。若數據中無明確日期,則詢問用戶確認。所有後續分析(包括「剛戰」的時間間距計算、新鮮度判斷、外部搜索日期限定)皆以此鎖定日期為基準。

### Step 0: 賽事級步速瀑布分析 [全場僅一次]

在分析任何馬匹前執行,產出共享上下文。

**步速類型:**

| 類型 | 定義 | 前領馬影響 |
|:---|:---|:---|
| 自殺式 | 前半程明顯快於班次標準 | 前領馬幾乎必定崩潰 |
| 真實 | 前半程略快於班次標準 | 平衡考驗,前領馬有機會撐住 |
| 中等 | 前半程接近班次標準 | 結果取決於個體能力,偏差小 |
| 龜速 | 前半程明顯慢於班次標準 | 前領馬偷襲成功率極高 |

**傳導鏈:**
1. 識別必須領放馬 (Compulsory Leaders) 數量
2. ≥ 3 匹搶位 → Genuine-to-Suicidal → 利後段馬
3. 僅 1 匹領放無挑戰 → Crawl → 放頭馬偷襲概率大增
3.5 **[長途中段突變 (Mid-Race Surge) 警告]:** 在 1800-2000m 賽事中,若預計步速為中等偏慢,但同場有 ≥2 匹「均速力拚」型馬匹於中前列,必須警惕中段發力 (踩油) 導致前領馬崩潰的可能性。
4. **[新增] 領放主導階梯 (Leader Dominance Hierarchy):** 恰好 2 匹潛在領放馬時,**不可**自動默認為搶位快步速。必須執行以下主導評估:
   - **因素 A — 路程專家 (Route Specialist):** 該馬是否在同場同程 ≥2 次領放勝出/上名?(連贏同程 = 極強訊號)
   - **因素 B — 近態強弱 (Current Form):** 近績是否在贏馬/穩定上名 vs 對方近績差/場地轉換/升班/首出?
   - **因素 C — 騎師權威 (Jockey Authority):** 騎師是否為當季 Top-5 且慣性控制步速 vs 對方騎師為被動型/見習?
   - **裁決:** 若一方在 ≥2/3 因素中明確佔優 → `DOMINANCE_GAP = 明確 (Clear)` → 預測 **Soft Lead / Crawl**(弱方高機率退讓,強方獲得輕鬆放頭)
   - **裁決:** 若雙方勢均力敵或各佔一項 → `DOMINANCE_GAP = 不明確 (Unclear)` → 維持 **Contested / Genuine** 預測(原始默認)
5. 預測崩潰點(通常 800-600m)
6. 後段馬需崩潰點前 200m 啟動

**共享輸出變數:**
- `PACE_TYPE` / `LEADER_COUNT` / `COLLAPSE_POINT` / `BIAS_DIRECTION` / `POSITION_MATRIX` / `DOMINANCE_GAP`

**場地配置修正:** 步速預測必須結合場地配置調整。詳見場地模組 (`resources/10a_track_sha_tin_turf.md`, `10b_track_happy_valley.md`, `10c_track_awt.md`):
- 沙田 C/C+3 Rail → 前領馬成功率上調,外疊蝕位加大
- 跑馬地 → 2 匹搶位 ≈ 沙田 3 匹效果(急彎加劇消耗)
- AWT → 前領馬成功率自動上調一級(泥地前速偏向)

**[SIP-ED1] 引擎-步速匹配矩陣 (Engine-Pace Matching Matrix):**

> [!IMPORTANT]
> Step 0 完成步速預測後,必須對每匹馬嘅引擎類型 (Step 2.6) 進行步速匹配判定。匹配結果直接傳入 Step 14.1 情境適配維度及 Step 14.2B 微調因素。

| 引擎類型 | 龜速 (Crawl) | 中等偏慢 | 中等 (Moderate) | 快 (Genuine) | 自殺式 (Suicidal) |
|:---|:---|:---|:---|:---|:---|
| **Type A (前領均速)** | ✅ 極度受惠 | ✅ 受惠 | ➖ 中性 | ⚠️ 體力風險 | ❌ 嚴重消耗 |
| **Type B (末段爆發)** | ❌ 嚴重受損 | ⚠️ 受損 | ➖ 中性 | ✅ 受惠 | ✅ 極度受惠 |
| **Type C (持續衝刺)** | ➖ 中性 | ✅ 受惠 | ✅ 受惠 | ➖ 中性 | ⚠️ 體力風險 |
| **Type A/B (混合)** | ➖ 視乎部署 | ➖ 中性 | ➖ 中性 | ➖ 中性 | ➖ 視乎位置 |

**匹配結果傳導:**
- ✅ 極度受惠/受惠 → 情境適配維度加分信號(見 Step 14.1)
- ❌ 嚴重受損/消耗 → 情境適配維度扣分信號 + Step 14.2B 偏差逆轉觸發
- ➖ 中性 → 不影響情境判定
- **LEADER_COUNT 修正:** Type A + 龜速 + LEADER_COUNT = 1(此馬為唯一前領)→ 匹配升至 ✅✅ 壟斷級受惠

### Step 1: 數據清洗與對齊
1. 讀取賽績字串,游標鎖定最左側 = 剛戰
2. `V` 或 Barrier Trial → 標記 `[近期試閘/事故]`,檢查是否 21 天內
3. **[賽績時效加權 (Form Recency Weighting)]:** 近六仗分析時,**剛戰權重最高**(核心錨定),前仗次之,大前仗作輔助參考,第 4-6 仗僅作背景趨勢參考。段速趨勢、EEM、賽績線驗證優先以最近 2-3 仗為判斷基準。

### Step 2: 情境交叉比對
產出情境標籤並傳入 Step 3:

| 情境 | 條件 | 處理 | 標籤 |
|:---|:---|:---|:---|
| A (隱藏價值) | 近績差但非今日場地/路程 | 回師首本 → 升級 | `[情境A-升級]` |
| B (虛火) | 近績好但非今日場地 | 降級/風險 | `[情境B-降級]` |
| C (正路) | 近績好 + 同程同場績優 | 穩膽 | `[情境C-正路]` |
| D (默認) | 近績中下 (6th-8th) + 同場同程仍差(無隱藏價值) | 無特別修正 | `[情境D-默認]` |
| D- (棄權級) | 近績大敗 (≥10th, 距離勝馬 ≥10L 連續出現) + 無任何寬恕因素 | 標記「冷門中的冷門」 | `[情境D⁻-棄權級]` |

### Step 2.5: [新增] 騎練數據分析

將 Data Retrieval 搜集的原始數據轉化為分析指標:
- **騎師適配度** (`JOCKEY_FIT`):硬朗型配被動馬 = 高;靈巧型配前速馬 = 高
- **組合上名率** (`COMBO_PLACE_RATE`):近 30 天統計,> 30% = 強勢組合
- **⚠️ 故障轉移:** 若外部搜索未能取得騎練組合數據 → `COMBO_PLACE_RATE = N/A`,評級矩陣中相關維度自動默認為 `➖ Neutral`,**嚴禁憑記憶捏造數字**。
- 供 Output Template Item 9 直接引用

### Step 2.6: [新增] 引擎類型分類 (Engine Type Classification)

> [!IMPORTANT]
> 此分類同步於 AU Wong Choi 引擎嘅 Type A/B/C 概念,但適配香港數據環境。每匹馬**必須**產出一個引擎類型判定,供 Output Template 嘅「引擎距離」欄位輸出及 Step 0 步速受惠/受損分析引用。

**引擎類型定義:**

| Type | 名稱 | 判定條件(主要:跑法模式 + 次要:段速驗證) | 步速適配 |
|:---|:---|:---|:---|
| **Type A** | 前領均速型 (Grinder) | 近5仗 ≥3仗 in-run 居前3位 + L600→L400 速差穩定(≤0.5s) | 需中等至快步速 |
| **Type B** | 末段爆發型 (Turn-of-Foot) | 近5仗 ≥3仗 in-run 居中後段 + L400 排全場前3 ≥2次 | 需遮擋、快步速利好 |
| **Type C** | 持續衝刺型 (Sustained Sprint) | 位置彈性大(前中後皆可)+ 末800m段速穩定無明顯衰退 | 需中等步速 |
| **Type A/B** | 混合型 | 無明確模式 / 處於跑法調整期 / 前中段均可 | 視乎部署 |

**判定流程(兩層驗證):**

1. **第一層:跑法模式(60% 權重)**
   - 從近5仗 in-running position 數據判斷馬匹慣常位置
   - 近5仗 ≥3仗居前3位 → 傾向 Type A
   - 近5仗 ≥3仗居中後段 → 傾向 Type B
   - 位置分散(前中後均勻混合)→ 傾向 Type C 或 Type A/B

2. **第二層:段速驗證(40% 權重)**
   - 「前領」但 L400 長期排全場尾段 → 降格為 Type A/B(偽前領)
   - 「後追」但 L400 長期排全場前列 → 確認 Type B(真爆發力)
   - 「後追」但 L400 排全場尾段 → 降格為 Type A/B(只係慢閘唔係爆發)

**距離適性表達(取代 Sire AWD):**

由於香港冇公開 Sire AWD 數據,距離適性**完全依靠馬匹自身歷史紀錄**:
- 計算馬匹各距離帶嘅 Win%/Place% (WR/PR)
- 標識最佳距離帶(WR/PR 最高嘅距離段)
- 初出馬(≤3場)可補充 Sire 距離傾向作輔助參考

**輸出格式:**
```
引擎距離:Type [A/B/C/A/B] ([中文標籤]) | 最佳 [距離帶] | 今仗[距離] = [場次:勝-亞-季] [✅/⚠️/❌]
```
示例:
- `引擎距離:Type A (前領均速型) | 最佳 1200-1400m | 今仗1600m = 3:0-1-0 ⚠️增程`
- `引擎距離:Type B (末段爆發型) | 最佳 1600-1800m | 今仗1650m = 5:2-1-1 ✅距離驗證`
- `引擎距離:Type A/B (混合型) | 最佳 1200m | 今仗1200m = 8:3-2-0 ✅`

**與其他 Steps 的聯動:**
- **Step 0 步速瀑布:** Type A → 慢步速受惠;Type B → 快步速受惠
- **Step 3 路程加權:** 引擎類型影響距離轉換風險(Type A 縮程 = 高風險;Type B 增程 = 需驗證 L400 衝力)
- **Step 10 段速法醫:** 引擎類型作為段速趨勢嘅解讀框架
- **Step 14 評級矩陣:** 引擎-步速匹配度影響「情境適配」維度

**接收 Step 2 情境修正:** A → 該路程/場地維度自動升為 ✅ Strong | B → 自動降為 ❌ Weak | C/D → 不修正

| 路程 | 優先因素 |
|:---|:---|
| ≥ 1800m | 氣量與狀態 |
| 1000-1400m | 時間與段速 |
| 1650m / HV | 班次與形勢 |

**新鮮度風險矩陣(傳入 Step 5 調整穩定性信心):**

| 間距 | 評估 |
|:---|:---|
| 21-45 天(短途) | ✅ 最佳 |
| 45-90 天(長途) | ✅ 最佳 |
| < 14 天 | ⚠️ 過密 |
| > 120 天 | ⚠️ 久休 |

> [!IMPORTANT]
> **蓄意出擊豁免 (Backup Intent Override):**
> 若馬匹觸發「過密 (<14天)」風險,但**同時觸發**了包含再出條件的「練馬師出擊訊號」(如 `Quick Backup`, `Tony Quick Backup` 等),則該練馬師的意圖 (Intent) **完全覆蓋**體力疑慮。強制解除 ⚠️ 過密 警告,定性為「蓄意部署」,在評級矩陣中其「路程/新鮮度」不可因此判為 ❌。
