## A. 格式完整性（最優先）

- [ ] 每匹馬都包含 11 個強制欄位？缺一即 FAIL
- [ ] 馬匹分析包含全部 10 項子欄位？
- [ ] 📊 評級矩陣包含全部 8 維度 + 寬恕加分 + 基礎評級 + 微調 + 覆蓋？

- [ ] D 級馬分析 ≥ 300 字且有具體「為何差」？
- [ ] 批次完成自檢行已輸出？

## C. SIP 合規檢查

> 合規 Agent 必須讀取 `00_sip_index.md`，逐條比對以下重點 SIP 是否被正確執行。

### SIP 觸發檢查矩陣
| SIP | 觸發條件 | 檢查重點 |
|:---|:---|:---|
| 練馬師訊號 | 當場有旗標練馬師 | 報告中有冇正確引用練馬師訊號？ |
| 品質守門員 (SIP-ST8) | 每批次 ≥6 匹累計 | Anti-Laziness 錨定有冇執行？ |
| 覆蓋規則鏈 | 永遠適用 | 風險封頂 > 溢價封頂 > 保底嘅優先級有冇被打亂？ |
| 場地模組 | 根據馬場 | 正確嘅場地模組有冇被載入？ |
| EEM 調整 | EEM 分析存在時 | EEM 判斷同步速假設有冇矛盾？ |

### 回歸舊習慣偵測 (Regression Detection)
最近 2 次覆盤中提出嘅 SIP，必須特別檢查是否已被套用：
- 若最近 SIP 涉及某個 Step/規則嘅修改 → 確認報告中有反映
- 若發現 Analyst 回歸到舊邏輯 → `[CRITICAL] SIP-REGRESSION`

## D. 反惰性深度審計

### D1. 跨批次品質一致性
- 比較每匹馬嘅分析字數 vs Batch 1 基線
- 若任何馬低於基線 70% → `[CRITICAL] MODEL-003`
- 若最後一批比第一批短 >40% → `[MINOR] MODEL-003`

### D2. 重複/幻覺偵測
- 重複結論：2+ 匹馬結論文字相似度 >80% → `[CRITICAL] LOGIC-003`
- 幻覺數據：2+ 匹馬出現完全相同統計數字 → `[CRITICAL] DATA-002`
- 評級矛盾：8 維度中 5+❌ 但評級 A+ → `[CRITICAL] LOGIC-001`

### D3. LLM 行為防護
- 截斷偵測：無 ✅ 批次完成行 → `[CRITICAL] MODEL-002`
- 指令遺忘偵測：後期批次省略早期有嘅區塊 → `[CRITICAL] MODEL-004`
- 格式漂移偵測：emoji/段落結構明顯改變 → `[MINOR] MODEL-005`

### D4. 批次隔離審計 (Batch Isolation Audit)
> 歷史教訓：當 Analyst 將多個批次合併寫入時，後期馬匹分析深度會大幅下降（低於 Batch 1 嘅 50%）。

**必檢項目：**
- [ ] **批次標記掃描**：搜索報告中嘅 `✅ 批次完成` 標記數量。預期數量 = `ceil(TOTAL_HORSES / 3)`。若標記數量少於預期 → `[CRITICAL] BATCH-001: 批次合併嫌疑`
- [ ] **品質斷崖偵測**：將所有馬匹按出現順序分成每 3 匹一組，計算每組嘅平均分析字數。若任何後期組別比第一組短 >40% → `[CRITICAL] BATCH-002: 品質斷崖`
- [ ] **批次間深度一致性**：比較第 1-3 匹馬 vs 第 4-6 匹馬 vs 第 7-9 匹馬嘅：
  - 段速法醫平均長度（行數）
  - EEM 分析平均長度（行數）
  - 評級矩陣每維度是否有獨立理據
  - 若後期組別嘅任何指標比第一組短 >50% → `[CRITICAL] BATCH-003: 後期批次偷工減料`

### D5. D/C 級馬反省略審計 (D-Horse Anti-Abbreviation Audit)

> **歷史教訓：** Analyst 最常見嘅偷懶行為係對 D/C-/C 級馬進行「預判省略」—— 因為預判評級低而壓縮分析，將本應為獨立段落嘅必填欄位（段速法醫、EEM、寬恕、14.2/14.2B/14.3）改為行內一句帶過、合併到評級矩陣、或完全省略。此行為係違反 Step B-HARD2 嘅直接體現。

**強制檢查（逐匹馬執行，不分評級）：**
- [ ] **獨立段落標題掃描**：對報告中嘅**每一匹馬**（包括 D 級），搜索以下 6 個必須存在嘅獨立段落標題/關鍵字：
  1. `段速法醫` 或 `🔬` — 必須為獨立段落（≥2 行），不可壓縮為評級矩陣行內
  2. `EEM` 或 `⚡` — 必須為獨立段落（≥2 行）
  3. `寬恕` 或 `📋` — 必須為獨立段落（≥1 行）
  4. `評級矩陣` 或 `📊` — 必須為獨立段落，7+ 維度各佔一行
  5. `14.2 基礎` + `14.2B` + `14.3 覆蓋` — 必須為 3 行獨立行
  6. `評語` 或 `💡` — 必須有優勢+劣勢+最大風險

- [ ] **若任何馬匹缺少上述任何一個 → `[CRITICAL] STRUCT-001: 馬[X] 缺少獨立段落 [段落名]`**
  - 嚴禁將此降級為 MINOR。缺少獨立段落 = 結構違規 = CRITICAL，無例外。
  - 嚴禁以「核心內容已在其他位置提及」為由豁免。每個段落必須為**物理獨立**嘅段落。

- [ ] **D/C 級馬特別字數檢查**：D/C-/C/C+ 級馬匹嘅分析字數必須 ≥300 字。若低於 → `[CRITICAL] STRUCT-002: 馬[X] D級分析不足300字`

- [ ] **D 級「為何差」具體性檢查**：D 級馬嘅分析中必須包含 ≥3 個具體數據點解釋「為何差」（例如具體名次、段速數字、穩定性百分比）。若僅有泛泛描述（如「表現差」「無走勢」）→ `[CRITICAL] STRUCT-003: 馬[X] D級分析欠缺具體數據`

> **執行優先級：** D5 審計嘅優先級等同 D1-D4。任何 D5 CRITICAL = 整場 FAIL。

## G. 定點掃描模式協議 (Targeted Rescan Mode Protocol)

> 當 Compliance Agent 收到 `RESCAN_MODE: TARGETED` 參數時，啟用此協議代替完整掃描。

### G1. 掃描範圍
- **只掃描** `REDO_BATCHES` 指定嘅已重做批次
- **只驗證** Top 4 排序同評級矩陣嘅一致性
- **跳過**未受影響嘅批次（信任前次通過結果）

### G2. 定點掃描檢查項
- [ ] 重做嘅批次通過 A-D 全部檢查？
- [ ] 重做嘅批次品質 ≥ BATCH_BASELINE × 70%？
- [ ] Top 4 排序同評級矩陣仍然一致？
- [ ] `PREVIOUS_ISSUES` 中列出嘅問題已全部修正？
- [ ] 重做內容同未改動內容之間冇格式/深度斷裂？

### G3. 結構性 vs 語義性 MINOR 分類標準

| 類型 | 定義 | 例子 | 處理方式 |
|:---|:---|:---|:---|
| **結構性 MINOR** | 格式/結構缺失，唔影響分析邏輯 | 缺段落標題、emoji 漏咗、14.2/14.2B 合併成一行 | 批次重做 + 定點掃描 |
| **語義性 MINOR** | 內容/邏輯問題，可能影響評級準確度 | 評級矛盾邊界 case、SIP 回歸嫌疑、交叉驗證失敗 | 歸類同 CRITICAL → 全場重做 |

**判斷原則：** 若有疑問，歸類為語義性（全場重做）。寧可過嚴，唔可漏放。
