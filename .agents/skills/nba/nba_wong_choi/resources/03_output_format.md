# NBA Wong Choi — 輸出格式定義

## 1. 逐場分析檔案
**檔名**:`Game_{GAME_TAG}_Full_Analysis.md`
**路徑**:`{TARGET_DIR}/Game_{GAME_TAG}_Full_Analysis.md`

**格式結構**（由上至下）:
1. **賽事背景** — 讓分/總分/獨贏/節奏/B2B/Blowout 風險
2. **傷病報告** — 逐隊列出傷兵及狀態
3. **新聞摘要** — NEWS_DIGEST
4. **Python Must-Respond 回應** — 回應 Python 前 5 名建議 (✅同意/⚡修改/❌拒絕)
5. **獨立發現** — LLM 自行發現嘅風險/機會 (至少 1 個)
6. **SGM Parlay 組合** — SGM 建議（≥2 組：🛡️ 穩膽 + 🔥 價值，可選 💎 進取）+ 條件觸發 Value Bomb

   **每個 Leg 必須包含以下欄位（數理引擎）：**
   - **賠率** (@X.XX)
   - **隱含勝率** (1/賠率 × 100%)
   - **預期勝率 / Adjusted Prob** (8-Factor 調整後，來自 Data Brief `adjusted_prob`)
   - **Edge** (預期勝率 - 隱含勝率)
   - **L10 命中** (X/10 = XX%)
   - **L10 逐場數組** (嚴禁省略)
   - **CoV** (穩定度評級: 🔒穩如磐石 / ✅低波幅 / ⚡中 / 🎲神經刀)
   - **8-Factor 明細** (trend/cov_adj/buffer/matchup/context/pace/usg/defender)

   **每個 Leg 必須包含以下欄位（邏輯引擎）：**
   - **核心邏輯** — LLM 原生撰寫，禁止照抄 Python
   - **對位優勢/劣勢**
   - **最大風險**
   - **克服信心**

   **每個組合必須包含組合結算：**
   - 總賠率 / 回報 / 組合 Edge / 注碼建議

   **組合結構強制要求**:
   - **組合 1:🛡️ 穩膽 SGM (Low Risk)** — L10 命中 ≥70%, 組合賠率 ≥ 2x
   - **組合 2:🔥 均衡 +EV 價值膽 (Mid Risk)** — 組合賠率 ≥ 3x, 目標 5x
   - **組合 3:💎 高倍率進取型 (High Risk)** — 組合賠率 ≥ 8x
   - **組合 X:💣 Value Bomb (條件觸發)** — Edge ≥10% 低估時出現

   > 賠率來自 Sportsbet 直接提取。組合賠率以 Sportsbet SGM 實際顯示為準。

7. **總結與賽前必做** — 最強/最弱關 + 賽前 60 分鐘必查

## 2. 最終報告

### 完整報告
**檔名**:`NBA_Analysis_Report.md`
**內容**:所有場次嘅完整分析(按賽程順序合併),包含:
- 賽程與背景總覽
- 傷病與剔除名單
- 每場 SGM 組合（≥2 組）
- 跨場次 Parlay 組合（如適用）
- 總結 + 信心評級 + 賽前 60 分鐘必做

### 穩膽精華報告
**檔名**:`NBA_Banker_Report.md`
**內容**:僅包含每場「組合 1:🛡️ 穩膽 SGM」嘅內容

## 3. 多場賽事合併協議
- 所有場次嘅分析**必須**合併寫入 `NBA_Analysis_Report.md`（按賽程順序）
- 嚴禁遺漏任何場次
- 每場分析之間用分隔線 `===` 隔開

## 4. 賽間自檢報告格式

### 通知用戶
每場賽事完成後:`「已完成 Game X ([Team A] vs [Team B]) 嘅數據提取同分析。」`

### 賽間問題匯報（按 `_session_issues.md` 內容）
- **有 CRITICAL** → 顯示簡述 + 建議 → 問用戶修正或跳過
- **只有 MINOR** → 每個問題一行 → 通知將於全部完成後統一處理
- **無問題** → 「Game [X] 分析完成,未發現異常。」

## 5. 最終匯報格式 (Step 5)
向用戶匯報:
1. **三組 Parlay 組合總覽**（簡表形式）
2. **最強組合推薦**
3. **賽前 60 分鐘必查事項**
4. **所有輸出檔案嘅絕對路徑**
