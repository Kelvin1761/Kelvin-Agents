# NBA Wong Choi — 輸出格式定義

## 1. 逐場分析檔案
**檔名**：`Game_[X]_[Teams]_Full_Analysis.txt`
**路徑**：`{TARGET_DIR}/Game_[X]_[Teams]_Full_Analysis.txt`

**格式結構**（由上至下）：
1. **NBA Meeting Intelligence** — 賽事背景、傷病、防守大閘狀態
2. **球員數據卡** — 每位候選球員嘅完整數據
3. **合格 Leg 候選清單** — 通過安全檢查嘅所有 Legs
4. **本場 3 組 Banker SGM 建議** — 穩健型 / 均衡型 / 進取型

## 2. 最終報告

### 完整報告
**檔名**：`NBA_Analysis_Report.txt`
**內容**：所有場次嘅完整分析（按賽程順序合併），包含：
- 賽程與背景總覽
- 傷病與剔除名單
- 防守大閘狀態
- 每場 3 組 SGM 組合
- 跨場次 3 組 Parlay 組合
- 總結 + 信心評級 + 賽前 60 分鐘必做

### 穩膽精華報告
**檔名**：`NBA_Banker_Report.txt`
**內容**：僅包含「組合 1：🛡️ 穩膽組合」嘅內容

## 3. 多場賽事合併協議
- 所有場次嘅分析**必須**合併寫入 `NBA_Analysis_Report.txt`（按賽程順序）
- 嚴禁遺漏任何場次
- 每場分析之間用分隔線 `===` 隔開

## 4. 賽間自檢報告格式

### 通知用戶
每場賽事完成後：`「已完成 Game X ([Team A] vs [Team B]) 嘅數據提取同分析。」`

### 賽間問題匯報（按 `_session_issues.md` 內容）
- **有 CRITICAL** → 顯示簡述 + 建議 → 問用戶修正或跳過
- **只有 MINOR** → 每個問題一行 → 通知將於全部完成後統一處理
- **無問題** → 「Game [X] 分析完成，未發現異常。」

## 5. 最終匯報格式 (Step 5)
向用戶匯報：
1. **三組 Parlay 組合總覽**（簡表形式）
2. **最強組合推薦**
3. **賽前 60 分鐘必查事項**
4. **所有輸出檔案嘅絕對路徑**
