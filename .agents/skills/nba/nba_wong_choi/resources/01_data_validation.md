# NBA Wong Choi — 數據品質驗證規則

> [!CAUTION]
> **Wong Choi 必須喺接收 Extractor 數據包後、傳畀 Analyst 之前，執行以下所有驗證。** 任何驗證失敗嘅數據必須要求 Extractor 重搜。

## 1. 數據包完整性檢查
- 確認每位候選球員都有完整 14 項數據卡（見 `nba_data_extractor/resources/02_data_card_template.md`）
- 若有缺失，要求 Extractor 補充

## 2. 跨場數據污染掃描 (Cross-Game Contamination Scan)
| 檢查項目 | 規則 | 處理 |
|:---|:---|:---|
| 球隊歸屬 | 每位球員必須屬於本場賽事嘅其中一隊 | 歸屬錯誤 = 串場污染 → 重搜 |
| L10 重複掃描 | 兩位不同球員嘅 L10 完全相同 | = 100% 幻覺 → 重搜 |
| 明星球員防呆 | All-Star 級球星核心數據低於合理門檻 | PTS<15 / AST<4 / REB<5 = 可疑 → 重搜 |
| Game 2+ 加強審查 | 第 2 場及以後嘅數據持更高懷疑態度 | 主動抽查驗證 |

## 3. L10 核對規則
- L10 第 1 場數據必須為「目標日期前最新一場」
- 若 L10 總和與賽季場均不符 → 重新檢索
- 10 個數據點必須來自 10 場獨立日期嘅比賽（防重複錄入）

## 4. 日期專精度檢查
- 確認 L10 數據第一項係該球員最新一場賽事
- 例如：分析 3/10 賽事，L10 第一項必須係 3/8 或 3/9 發生嘅比賽
- L10 第一場嘅「日期」與「對手」必須 100% 準確

## 5. 比分板核對 (Box Score Integrity)
- **必須**先檢查前一日嘅全聯盟比分板
- 若球員喺前一日有上場紀錄且表現突出（例如 25+ 分或三雙），數據卡**必須**包含該場次
- 若傷病報告話佢 Out，但佢琴日打咗 → 視為 Active

## 6. 數據源衝突處理
- 若不同來源數據不一 → 優先採用與用戶已開檔案一致嘅賽程
- 若用戶已有分析檔案（如 `Game_1_CLE_vs_ORL`），以該檔案嘅賽程為準

## 7. 用戶回報錯誤處理
若用戶回報數據錯誤，執行 `deep_verify` 程序：
1. 對比 Box Score
2. 對比 L10 Game Log
3. 對比最新傷病動態
4. 修正後重新輸出
