# 球員數據卡格式 (Player Data Card Template)

本文件定義每位候選球員必須填寫嘅數據卡格式。所有 14+1 項欄位必須完整填充,嚴禁省略。

---

## 數據卡格式

```
--- 球員數據卡 ---

| # | 項目 | 數值 | 來源 |
|---|------|------|------|
| 1 | 球員姓名 / 現役球隊 | [已核實] | [來源] |
| 2 | QA 雙重驗證印記 | [✅/⚠️/❌ — 詳見 01_data_protocols.md] | [內部檢核] |
| 3 | L10 逐場數據 | [最新一場, 第二新, ..., 最舊一場] (僅數值,由左至右為新到舊) | [來源] |
| 4 | 未達標原因分析 | [分析 L10 中未過線場次嘅原因,情有可原?事實描述,不做判斷] | [來源] |
| 5 | 場均上場時間 (L5) | [分鐘] | [來源] |
| 6 | 主/客場、H2H 場均 | [主場均值 / 客場均值 / H2H 均值] | [來源] |
| 7 | 對手防守排名 (DvP) | 第 X/30 | [來源] |
| 8 | 對位防守影響 | [大個/後衛/禁區威懾 — 對手姓名與具體壓制描述] | [來源] |
| 9 | 得分類型 | [切入型 (≥40%禁區) / 投射型 / 混合] | [來源] |
| 10 | 節奏 / B2B / 休息天數 | [高/中/低] / [是/否] / [X天] | [來源] |
| 11 | 垃圾時間風險 | [無/中等/高/極高] (基於讓分盤) | [來源] |
| 12 | 莊家誘餌/聰明錢 | [上升/下降/異常誘餌/N/A] | [來源] |
```

## 填寫規則

### 必填項 (所有 15 項)
- 每一項都必須填寫,缺失數據標記為 `N/A (數據不足)`。
- 嚴禁使用 `...`、`[同上]`、`[略]` 等省略語。

### L10 逐場數據 (項目 3)
- 必須列出完整 10 場數據,由最新到最舊排列。
- 格式:`[28, 25, 31, 18, 22, 30, 27, 24, 29, 26]`
- 若球員近期場次不足 10 場,列出所有可用場次並註明。
- 數據類型取決於盤口:得分盤用 PTS、籃板盤用 REB、助攻盤用 AST。
- **若球員同時有多個盤口候選**,需為每個盤口類型分別列出 L10。

### 未達標原因分析 (項目 4)
- 檢視 L10 中表現低於盤口線嘅場次。
- 記錄客觀原因:對手強弱、上場時間異常、犯規麻煩、提前退場等。
- **嚴禁做判斷** — 只記錄事實。

### 對位防守影響 (項目 8)
- 必須根據 `03_defensive_profiles.md` 嘅分類進行對位匹配。
- 格式:`外線鎖死型 — Jrue Holiday (Active) — 場均 DRTG 105`
- 若對位防守者缺陣,標明 `[Out]` 並標記為潛在正面情境。

### 近期新聞摘要 (項目 15)
- 記錄過去 72 小時內與該球員相關嘅重要新聞。
- 每條新聞格式:`[正面/負面/中性] 新聞摘要 (來源, 日期)`
- 涵蓋:傷病更新、狀態趨勢、角色變更、場外事件、教練評價。
- 若無重要新聞,填寫 `無近期重要新聞`。

### 垃圾時間風險 (項目 11)
根據讓分盤判定:
| 讓分盤 | 風險等級 |
|--------|----------|
| < 7.5 | 無 |
| 7.5 - 10 | 中等 |
| 10.5 - 14 | 高 |
| 15+ | 極高 |

## 候選球員篩選標準

Extractor 應優先提取以下類型球員嘅數據:
1. **高使用率球星** — 場均 ≥ 28 分鐘上場時間
2. **極度穩定角色球員** — 近期表現波動小
3. **傷缺紅利受益者** — 隊友傷缺導致球權/上場時間增加
4. **對位優勢球員** — 對手防守弱點明顯

應排除:
- 已確認缺陣 (Out) 嘅球員
- 上場時間 L5 < 20 分鐘嘅球員
- 被標記為「打爆風險極高」場次嘅邊緣球員
