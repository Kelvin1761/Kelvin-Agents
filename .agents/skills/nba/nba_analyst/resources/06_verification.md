# 輸出前自檢清單 (Pre-Output Verification Checklist)

在生成最終報告之前，必須逐項核對以下清單。任何未通過嘅項目必須修正後再輸出。

---

## 自檢清單

### 數據完整性
- [ ] 所有球員已確認在 Active Roster（非已交易/傷病 Out）？
- [ ] L10 數據已通過多個可靠數據源交叉比對？（來自 Extractor 數據包）
- [ ] 所有盤口數字符合 **Bet365 嚴格選項規則 (Strict Built-in Matrix)**？絕無自創中間數字 (如 22.5)？
- [ ] 無孤立判斷（每個結論至少 2 個數據點支持）？

### 組合完整性（匯總模式）
- [ ] 全部 3 個組合已輸出（🛡️ 穩膽 / 💎 價值 / 🔥 高賠）？
- [ ] 每個 Leg 都有完整數據欄位？
  - 首次出現嘅球員：完整 5 項數據子欄位？
  - 重複出現嘅球員：有引用 + 獨立分析（命中率/+EV/核心邏輯/風險）？
- [ ] 所有組合都有 📊 組合分析區塊？
- [ ] +EV 篩選已為每個 Leg 填寫（包括組合 2 和 3 嘅所有 Legs）？

### 安全性
- [ ] 打爆風險賽事已正確標記/避開？
- [ ] SGP 相關性已審查（無互相蠶食/劇本矛盾/天花板衝突）？
- [ ] 致命缺陷球員已剔除？
- [ ] 命中率底線已審查（穩膽 ≥80% / 價值 ≥75%）？

### 防守對位
- [ ] 每位候選球員都有位置對位防守分析？
- [ ] 候選球員嘅得分類型已標註（切入型/投射型）？
- [ ] 防守調整值已正確套用？
- [ ] 鐵桶陣/禁區真空情境已識別？

### 新聞情境
- [ ] Extractor 提供嘅新聞摘要已被納入分析？
- [ ] 重要負面新聞已反映在情境調整扣分中？
- [ ] 核心邏輯中有提及相關新聞影響？
- [ ] NEWS_DIGEST 已整合入報告頭部？

### 反惰性
- [ ] 組合 3 (🔥 高賠型) 嘅分析深度與組合 1 一致？
- [ ] 無任何 `...`、`[同上]`、`[略]` 等省略語？
- [ ] 全篇完整性檢查：所有 Leg 分析、核心邏輯與數據列表前後一致且未中斷？

---

## 修正機制

若發現任何項目未通過：
1. 標記具體問題所在
2. 回到對應 Step 補充缺失內容
3. 重新執行自檢
4. 所有項目通過後才可輸出最終報告

---

## 品質標準

| 等級 | 通過率 | 評價 |
|------|--------|------|
| ✅ 優秀 | 100% 全部通過 | 可輸出 |
| ⚠️ 合格 | ≥ 90% 通過，缺失為非關鍵項 | 標註缺失後可輸出 |
| ❌ 不合格 | < 90% 通過 | 必須修正後再輸出 |
