# 輸出前自檢清單 (Pre-Output Verification Checklist)

在生成最終報告之前,必須逐項核對以下清單。任何未通過嘅項目必須修正後再輸出。

---

## 自檢清單

### 數據完整性
- [ ] 所有球員已確認在 Active Roster(非已交易/傷病 Out)?
- [ ] L10 數據已通過多個可靠數據源交叉比對?(來自 Extractor 數據包)
- [ ] 所有盤口數字符合 **Bet365 嚴格選項規則 (Strict Built-in Matrix)**?絕無自創中間數字 (如 22.5)?
- [ ] 無孤立判斷(每個結論至少 2 個數據點支持)?

### 組合完整性
- [ ] 全部 **4 個組合**已輸出（🛡️ 1A + 🔥 1B + 🔥 2 + 💎 3）?
- [ ] 每個 Leg 都有完整 `[FILL]` 骨架欄位（參照 `05_output_template.md`）?
  - 數理引擎 + 邏輯引擎表格?
  - +EV 篩選?
  - 數據卡（L10 逐場數組 + 未達標剖析 + 均值/中位/SD/CoV）?
  - 進階數據（USG%/TS%/Pace-Adj）?
  - 防守對位?
  - 場景分裂（主客/Rest/H2H）?
  - 情境調整值?
- [ ] 所有組合都有 📊 組合分析區塊?
- [ ] +EV 篩選已為每個 Leg 填寫（包括組合 1B、2 和 3 嘅所有 Legs）?
- [ ] `[FILL]` 殘留數 = 0（無任何未填寫嘅佔位符）?

### 安全性
- [ ] 打爆風險賽事已正確標記/避開?
- [ ] SGP 相關性已審查(無互相蠶食/劇本矛盾/天花板衝突)?
- [ ] 致命缺陷球員已剔除?
- [ ] 命中率底線已審查(穩膽 ≥80% / 價值 ≥75%)?

### 防守對位
- [ ] 每位候選球員都有位置對位防守分析?
- [ ] 候選球員嘅得分類型已標註(切入型/投射型)?
- [ ] 防守調整值已正確套用?
- [ ] 鐵桶陣/禁區真空情境已識別?

### 新聞情境
- [ ] Extractor 提供嘅新聞摘要已被納入分析?
- [ ] 重要負面新聞已反映在情境調整扣分中?
- [ ] 核心邏輯中有提及相關新聞影響?
- [ ] NEWS_DIGEST 已整合入報告頭部?

### 反惰性
- [ ] 組合 1B、2、3 嘅分析深度與組合 1A 一致?
- [ ] 無任何 `...`、`[同上]`、`[略]`、`[參見組合X]`、`[📋 完整數據見組合 X]` 等省略語?
- [ ] 全篇完整性檢查:所有 Leg 分析、核心邏輯與數據列表前後一致且未中斷?
- [ ] `[FILL]` 殘留掃描：輸出中 `[FILL]` 出現次數 = 0?

---

## 修正機制

若發現任何項目未通過:
1. 標記具體問題所在
2. 回到對應 Step 補充缺失內容
3. 重新執行自檢
4. 所有項目通過後才可輸出最終報告
