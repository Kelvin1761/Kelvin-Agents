# 輸出前自檢清單 (Pre-Output Verification Checklist) — V2.0

在生成最終報告之前,必須逐項核對以下清單。任何未通過嘅項目必須修正後再輸出。

---

## 自檢清單

### 數據完整性
- [ ] 所有球員已確認在 Active Roster（非已交易/傷病 Out）?
- [ ] L10 數據來自 Extractor 數據包（nba_api + ESPN）?
- [ ] 所有盤口數字來自 **Bet365 直接提取**（Claw V6 / MCP Playwright）?
- [ ] 無孤立判斷（每個結論至少 2 個數據點支持）?

### 組合完整性
- [ ] **≥2 個組合**已輸出（🛡️ 1 + 🔥 2，可選 💎 3）? 組合 X 為條件觸發（可選）?
- [ ] 每個 Leg 都有完整分析欄位?
  - 數理引擎 + 邏輯引擎表格?
  - Bet365 盤口對照表?
  - 數據卡（L10 逐場數組 + 均值/中位/SD/CoV）?
  - 進階數據（USG%/Pace-Adj/Adjusted Win Prob）?
  - 防守對位?
  - 情境調整（8-Factor 明細）?
- [ ] 所有組合都有 📊 組合結算區塊?
- [ ] `[FILL]` 殘留數 = 0（無任何未填寫嘅佔位符）?
- [ ] Python 自動核心邏輯已由 Analyst 審閱並補充深度分析?

### 安全性
- [ ] 打爆風險賽事已正確標記/避開?
- [ ] SGP 相關性已審查（無互相蠶食/劇本矛盾/天花板衝突）?
- [ ] 致命缺陷球員已剔除（傷兵 Out 狀態）?
- [ ] 組合間球員分散規則已遵守?

### 防守對位
- [ ] 每位候選球員都有位置對位防守分析?
- [ ] 候選球員嘅得分類型已標註（切入型/投射型）?
- [ ] 防守調整值已正確反映喺 8-Factor 引擎中?

### 新聞情境
- [ ] Extractor 提供嘅新聞摘要已被納入分析?
- [ ] 重要負面新聞已反映在情境調整扣分中?
- [ ] NEWS_DIGEST 已整合入報告頭部?

### 反惰性（Anti-Laziness）
- [ ] 組合 2、3 嘅分析深度與組合 1 一致?
- [ ] 無任何 `...`、`[同上]`、`[略]`、`[參見組合X]`、`[📋 完整數據見組合 X]` 等省略語?
- [ ] 全篇完整性檢查:所有 Leg 分析、核心邏輯與數據列表前後一致且未中斷?
- [ ] `[FILL]` 殘留掃描：輸出中 `[FILL]` 出現次數 = 0?
- [ ] Analyst 為每個 Leg 嘅 Python 自動邏輯補充咗獨立深度分析?

### Python ↔ LLM 協作品質
- [ ] Analyst 有閱讀 Python 嘅 `📐 Python 推理` 區塊並理解 8-Factor 調整依據?
- [ ] Analyst 嘅補充分析有加入 Python 無法量化嘅角度（比賽劇本推演、教練策略、Chemistry）?
- [ ] 若 Analyst 唔同意 Python 嘅結論，已明確標註分歧並解釋原因?

---

## 修正機制

若發現任何項目未通過:
1. 標記具體問題所在
2. 回到對應 Step 補充缺失內容
3. 重新執行自檢
4. 所有項目通過後才可輸出最終報告
