### B2. 全場決策與分析陷阱格式合規 (Verdict Format Compliance)

> **歷史教訓:** 模型傾向省略 Part 3/4 嘅完整格式,將 Top 4 格式簡化(缺少 `**馬號及馬名:**` 等標籤行),或完全省略 Part 4 嘅必填段落(穩建馬外報、冷門馬總計等)。

**Part 3 必檢項:**
- [ ] `Speed Map 回顧` 行存在?(必須在 Top 4 之前)→ 缺少 = `[MINOR] FORMAT-P3-001`
- [ ] `Top 4 位置精選` 標題使用正確全名?(唔係 `Top 4 精選`)→ 格式偏離 = `[MINOR] FORMAT-P3-002`
- [ ] Top 4 每位精選有 4 項獨立標籤行:`**馬號及馬名:**`、`**評級與✅數量:**`、`**核心理據:**`、`**最大風險:**` → 缺少任何標籤 = `[MINOR] FORMAT-P3-003`
- [ ] `🎯 Top 2 入三甲信心度` 使用完整標題?→ 缺少 = `[MINOR] FORMAT-P3-004`

**Part 4 必檢項:**
- [ ] `## [第四部分] 分析陷阱` 標題存在?→ 缺少 = `[MINOR] FORMAT-P4-001`
- [ ] 市場預期警告 + 步速逆轉保險 + **整體潛在機會建議** 三項齊全?→ 缺少任何 = `[MINOR] FORMAT-P4-002`
- [ ] `🚨 緊急煞車檢查` 段落存在?→ 缺少 = `[MINOR] FORMAT-P4-003`
- [ ] `📊 穩建馬外報建議` 段落存在?→ 缺少 = `[MINOR] FORMAT-P4-004`
- [ ] `🐴⚡ 冷門馬總計` 段落存在?→ 缺少 = `[MINOR] FORMAT-P4-005`

## D. 反惰性深度審計

### D1. 跨批次品質一致性
- 字數 vs 基線偏差 >30% → `[CRITICAL] MODEL-003`
- 最後一批比第一批短 >40% → `[MINOR] MODEL-003`

### D2. 重複/幻覺偵測
- 結論相似度 >80% → `[CRITICAL] LOGIC-003`
- 完全相同統計數字 → `[CRITICAL] DATA-002`
- 矩陣矛盾 → `[CRITICAL] LOGIC-001`

### D3. LLM 行為防護
- 截斷 → `[CRITICAL] MODEL-002`
- 指令遺忘 → `[CRITICAL] MODEL-004`
- 格式漂移 → `[MINOR] MODEL-005`

### D4. 批次隔離審計 (Batch Isolation Audit)
> 歷史教訓:當 Analyst 將多個批次合併寫入時,後期馬匹分析深度會大幅下降。

**必檢項目:**
- [ ] **批次標記掃描**:搜索 `✅ 批次完成` 標記數量。預期 = `ceil(TOTAL_HORSES / 3)`。標記不足 → `[CRITICAL] BATCH-001`
- [ ] **品質斷崖偵測**:每 3 匹一組計算平均字數。後期組 <第一組 60% → `[CRITICAL] BATCH-002`
- [ ] **批次間深度一致性**:比較各組嘅段速/EEM/矩陣分析長度。後期組任何指標 <第一組 50% → `[CRITICAL] BATCH-003`

### D5. D/C 級馬反省略審計 (D-Horse Anti-Abbreviation Audit)

> **歷史教訓:** Analyst 最常見嘅偷懶行為係對 D/C-/C 級馬進行「預判省略」—— 因為預判評級低而壓縮分析,將本應為獨立區塊嘅必填欄位改為行內一句帶過、合併到評級矩陣、或完全省略。

**強制檢查(逐匹馬執行,不分評級):**
- [ ] **獨立區塊掃描**:對報告中嘅**每一匹馬**(包括 D 級),搜索以下 5 個必須存在嘅獨立區塊:
  1. `⏱️ 近績解構` — 必須為獨立區塊
  2. `🐴 馬匹剖析` — 必須包含 5 項子欄位(班次負重+引擎距離+步態場地+配備意圖+人馬組合)
  3. `🧠 核心分析推演` — 必須包含 4 項(EEM+段速+陣型+風險),各為獨立段落
  4. `📊 評級矩陣` — 必須為獨立段落,8 維度各佔一行
  5. `💡 結論` — 必須有優勢+劣勢+最大失敗風險+最終評級

- [ ] **若任何馬匹缺少上述任何一個 → `[CRITICAL] STRUCT-001: 馬[X] 缺少獨立區塊 [區塊名]`**
  - 嚴禁將此降級為 MINOR。缺少獨立區塊 = 結構違規 = CRITICAL,無例外。
  - 嚴禁以「核心內容已在其他位置提及」為由豁免。每個區塊必須為**物理獨立**嘅區塊。

- [ ] **D/C 級馬特別字數檢查**:D/C-/C/C+ 級馬匹嘅分析字數必須 ≥300 字。若低於 → `[CRITICAL] STRUCT-002: 馬[X] D級分析不足300字`

- [ ] **D 級「為何差」具體性檢查**:D 級馬嘅分析中必須包含 ≥3 個具體數據點解釋「為何差」(例如具體名次、段速數字、穩定性百分比)。若僅有泛泛描述 → `[CRITICAL] STRUCT-003: 馬[X] D級分析欠缺具體數據`

> **執行優先級:** D5 審計嘅優先級等同 D1-D4。任何 D5 CRITICAL = 整場 FAIL。

## G. 定點掃描模式協議 (Targeted Rescan Mode Protocol)

> 當 Compliance Agent 收到 `RESCAN_MODE: TARGETED` 參數時,啟用此協議代替完整掃描。

### G1. 掃描範圍
- **只掃描** `REDO_BATCHES` 指定嘅已重做批次
- **只驗證** Top 3 排序同評級矩陣嘅一致性
- **跳過**未受影響嘅批次(信任前次通過結果)

### G2. 定點掃描檢查項
- [ ] 重做嘅批次通過 A-D 全部檢查?
- [ ] 重做嘅批次品質 ≥ BATCH_BASELINE × 70%?
- [ ] Top 3 排序同評級矩陣仍然一致?
- [ ] `PREVIOUS_ISSUES` 中列出嘅問題已全部修正?
- [ ] 重做內容同未改動內容之間冇格式/深度斷裂?

### G3. 結構性 vs 語義性 MINOR 分類標準

| 類型 | 定義 | 例子 | 處理方式 |
|:---|:---|:---|:---|
| **結構性 MINOR** | 格式/結構缺失,唔影響分析邏輯 | 缺區塊標題、emoji 漏咗、子欄位合併 | 批次重做 + 定點掃描 |
| **語義性 MINOR** | 內容/邏輯問題,可能影響評級準確度 | 評級矛盾邊界 case、SIP 回歸嫌疑、交叉驗證失敗 | 歸類同 CRITICAL → 全場重做 |

**判斷原則:** 若有疑問,歸類為語義性(全場重做)。寧可過嚴,唔可漏放。
