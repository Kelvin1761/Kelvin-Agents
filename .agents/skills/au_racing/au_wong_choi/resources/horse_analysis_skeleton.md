# 馬匹分析骨架模板（Per-Horse Skeleton — AU Version V2）

> **用法:** Wong Choi 喺每個 Batch 開始前,將此骨架 × BATCH_SIZE 注入到 Analyst prompt。
> LLM 嘅任務係「填充骨架」而唔係「生成分析」,保證 100% 結構完整性。
> **核心邏輯/結論部分為 LLM 自由發揮區域** — 結構固定但分析深度同角度由 LLM 自由發揮。

> ⚠️ **P39 V2 Python-Driven Fact Anchors (2026-04-08 升級):**
> Python `inject_fact_anchors.py` V2 已預填以下所有數據區塊：
> - `📌 Racecard 事實錨點` — Last 10 / Career / Track+Dist stats / Surface stats / Cycle
> - `📋 完整賽績檔案` — 近 10 場（正式🏁 + 試閘🔰 混合），每場含 走位形勢分析
> - `📊 段速法醫` — PI 位移指數趨勢 + L400 PI 趨勢
> - `⚡ 形勢與走位檔案` — 累積消耗加權評分
> - `🔧 引擎距離` — Engine Type A/B/C/AB + 距離適性 W-P-L
>
> **LLM 嚴禁修改以上數值。LLM 嘅工作只係「解讀」同「判斷」。**
>
> **Last 10 String 解讀規則:**
> 澳洲格式 `Last 10` 從**左到右 = 最新到最舊**。
> - 數字 1-9 = 該場最終名次
> - `0` = **第 10 名**（唔係 0 分或 null）
> - `x` = Trial（試閘）或 Scratched（退出），跳過不計
