## Internal Tracking 演算法（不可見的核心運算）

**以下步驟純屬內部運算，絕對禁止將過程印在最終畫面上。每批馬完成後，直接輸出其對應的 [第二部分] 結果。**

### 📅 賽事日期鎖定 (Race Date Lock) [最優先執行]
從用戶提供的原始數據中識別並鎖定今日賽事日期。若數據中無明確日期，則詢問用戶確認。所有後續分析（包括「剛戰」的時間間距計算、新鮮度判斷、外部搜索日期限定）皆以此鎖定日期為基準。

### Step 0: 賽事級步速瀑布分析 [全場僅一次]

在分析任何馬匹前執行，產出共享上下文。

**步速類型：**

| 類型 | 定義 | 前領馬影響 |
|:---|:---|:---|
| 自殺式 | 前半程明顯快於班次標準 | 前領馬幾乎必定崩潰 |
| 真實 | 前半程略快於班次標準 | 平衡考驗，前領馬有機會撐住 |
| 中等 | 前半程接近班次標準 | 結果取決於個體能力，偏差小 |
| 龜速 | 前半程明顯慢於班次標準 | 前領馬偷襲成功率極高 |

**傳導鏈：**
1. 識別必須領放馬 (Compulsory Leaders) 數量
2. ≥ 3 匹搶位 → Genuine-to-Suicidal → 利後段馬
3. 僅 1 匹領放無挑戰 → Crawl → 放頭馬偷襲概率大增
3.5 **[長途中段突變 (Mid-Race Surge) 警告]：** 在 1800-2000m 賽事中，若預計步速為中等偏慢，但同場有 ≥2 匹「均速力拚」型馬匹於中前列，必須警惕中段發力 (踩油) 導致前領馬崩潰的可能性。
4. **[新增] 領放主導階梯 (Leader Dominance Hierarchy)：** 恰好 2 匹潛在領放馬時，**不可**自動默認為搶位快步速。必須執行以下主導評估：
   - **因素 A — 路程專家 (Route Specialist)：** 該馬是否在同場同程 ≥2 次領放勝出/上名？（連贏同程 = 極強訊號）
   - **因素 B — 近態強弱 (Current Form)：** 近績是否在贏馬/穩定上名 vs 對方近績差/場地轉換/升班/首出？
   - **因素 C — 騎師權威 (Jockey Authority)：** 騎師是否為當季 Top-5 且慣性控制步速 vs 對方騎師為被動型/見習？
   - **裁決：** 若一方在 ≥2/3 因素中明確佔優 → `DOMINANCE_GAP = 明確 (Clear)` → 預測 **Soft Lead / Crawl**（弱方高機率退讓，強方獲得輕鬆放頭）
   - **裁決：** 若雙方勢均力敵或各佔一項 → `DOMINANCE_GAP = 不明確 (Unclear)` → 維持 **Contested / Genuine** 預測（原始默認）
5. 預測崩潰點（通常 800-600m）
6. 後段馬需崩潰點前 200m 啟動

**共享輸出變數：**
- `PACE_TYPE` / `LEADER_COUNT` / `COLLAPSE_POINT` / `BIAS_DIRECTION` / `POSITION_MATRIX` / `DOMINANCE_GAP`

### Step 1: 數據清洗與對齊
1. 讀取賽績字串，游標鎖定最左側 = 剛戰
2. `V` 或 Barrier Trial → 標記 `[近期試閘/事故]`，檢查是否 21 天內
3. **[賽績時效加權 (Form Recency Weighting)]：** 近六仗分析時，自動適用時效加權：剛戰 ×3 | 前仗 ×2 | 大前仗 ×1.5 | 第4-6仗 ×1。此加權影響段速趨勢、EEM 評估及賽績線驗證的判斷權重。

### Step 2: 情境交叉比對
產出情境標籤並傳入 Step 3：

| 情境 | 條件 | 處理 | 標籤 |
|:---|:---|:---|:---|
| A (隱藏價值) | 近績差但非今日場地/路程 | 回師首本 → 升級 | `[情境A-升級]` |
| B (虛火) | 近績好但非今日場地 | 降級/風險 | `[情境B-降級]` |
| C (正路) | 近績好 + 同程同場績優 | 穩膽 | `[情境C-正路]` |
| D (默認) | 近績中下 (6th-8th) + 同場同程仍差（無隱藏價值） | 無特別修正 | `[情境D-默認]` |
| D- (棄權級) | 近績大敗 (≥10th, 距離勝馬 ≥10L 連續出現) + 無任何寬恕因素 | 標記「冷門中的冷門」 | `[情境D⁻-棄權級]` |

> [!IMPORTANT]
> 若 Step 12 結論為「不可作準」→ 回溯此步驟，以最近可作準賽事重新錨定。

### Step 2.5: [新增] 騎練數據分析

將 Data Retrieval 搜集的原始數據轉化為分析指標：
- **騎師適配度** (`JOCKEY_FIT`)：硬朗型配被動馬 = 高；靈巧型配前速馬 = 高
- **組合上名率** (`COMBO_PLACE_RATE`)：近 30 天統計，> 30% = 強勢組合
- **⚠️ 故障轉移：** 若外部搜索未能取得騎練組合數據 → `COMBO_PLACE_RATE = N/A`，評級矩陣中相關維度自動默認為 `➖ Neutral`，**嚴禁憑記憶捏造數字**。
- 供 Output Template Item 9 直接引用

### Step 3: 全方位加權

**接收 Step 2 情境修正：** A → 該路程/場地維度自動升為 ✅ Strong | B → 自動降為 ❌ Weak | C/D → 不修正

| 路程 | 優先因素 |
|:---|:---|
| ≥ 1800m | 氣量與狀態 |
| 1000-1400m | 時間與段速 |
| 1650m / HV | 班次與形勢 |

**新鮮度風險矩陣（傳入 Step 5 調整穩定性信心）：**

| 間距 | 評估 |
|:---|:---|
| 21-45 天（短途） | ✅ 最佳 |
| 45-90 天（長途） | ✅ 最佳 |
| < 14 天 | ⚠️ 過密 |
| > 120 天 | ⚠️ 久休 |

> [!IMPORTANT]
> **蓄意出擊豁免 (Backup Intent Override)：** 
> 若馬匹觸發「過密 (<14天)」風險，但**同時觸發**了包含再出條件的「練馬師出擊訊號」（如 `Quick Backup`, `Tony Quick Backup` 等），則該練馬師的意圖 (Intent) **完全覆蓋**體力疑慮。強制解除 ⚠️ 過密 警告，定性為「蓄意部署」，在評級矩陣中其「路程/新鮮度」不可因此判為 ❌。

### Step 4: 溢價校正 (Premium Jockey Correction)

> [!WARNING]
> **[HKJC ENGINE ONLY]** 本 Step 4 嘅騎師溢價校正（潘頓、莫雷拉、跑馬地規則等）僅適用於 HKJC 引擎。
> **AU 引擎嘅騎練訊號見 `au_horse_analyst/resources/02e_jockey_trainer.md`。**

> [!IMPORTANT]
> **邏輯前提：** Z. Purton 潘頓是香港頂級騎師，其座騎通常被 AI 過度追捧，導致分析偏離實際勝出機率。AI 必須主動抑制對潘頓的盲目崇拜，避免產出過度樂觀的評級。
> **適用範圍 (Market Darling Tag)：** 除潘頓外，以下騎師回港客串或策騎受注目馬時，**自動觸發相同校正流程**：莫雷拉 (J. Moreira)、麥道朗 (J. McDonald)、或任何當季 Top-3 冠軍騎師排名的騎師。觸發條件與潘頓完全一致。

**校正規則（分兩階段執行）：**

**階段一：旗標設定（思考過程中執行，於 Step 4 位置）：**

| 規則 | 觸發條件 | 處理方式 |
|:---|:---|:---|
| **1. 門檻提高 (High Bar)** | 騎師為潘頓或觸發 Market Darling Tag | 設定旗標：該馬必須具備「實質賽績優勢」才可給予 S 或 A。所謂實質優勢 = 至少一項：段速趨勢上升軌、上仗明顯受阻（寬恕檔案有 ≥ 2 因素）、超級降班馬、或強組出品。僅靠「穩定性」一項不足以支持 S/A。 |
| **1b. 3歲谷草新手加強版 (3YO HV Debut Elevated Bar)** | 旗標已設 + 馬匹為 3 歲 + 跑馬地出賽 ≤2 次 | 門檻進一步提高：必須具備 **≥2 個核心 ✅** 才可給予 A 級；否則最高封頂 **B+**。跑馬地緊湊彎道、夜賽氣氛、獨特的步速動態對 3 歲新馬是巨大未知數，即使配載頂級騎師亦不可盲目信任。 |
| **2. 嚴禁純粹加分 (No Free Upgrade)** | 近績平平（如 5-6-7 名）且無寬恕因素 + 騎師觸發旗標 | **嚴禁** 因換上明星騎師而直接升級。必須假設此為「市場陷阱」或「過度受注目」。評級維持原有維度判斷結果，不因騎師身份額外加分。 |

**階段二：強制執行（在 Step 14.1 完成維度判斷後執行）：**

| 規則 | 觸發條件 | 處理方式 |
|:---|:---|:---|
| **3. 價值陷阱偵測 (Value Trap)** | 旗標已設 + 評級矩陣 ✅ ≤ 2 | 標記為 **`[高風險受注目馬 (Risky Favorite)]`**，並在評語中明確警告。 |
| **4. 反向搜索 (Reverse Check)** | 旗標騎師座騎最終評為 B 或以下 | 檢查是否存在被市場忽略的非旗標騎師馬匹反而具備更強的 ✅ 數量。若有，在評語中標註「比明星騎師座騎更具潛在價值」。 |

**與 Step 14 的互動：**
- 旗標已設 + 評級矩陣中「段速」和「情境」均非 ✅ → 強制封頂 B（覆蓋規則已在 Step 14.3 定義）
- 若旗標騎師座騎確實具備實質優勢（多個 ✅ 且包含段速或情境）→ 正常評級，不受校正影響

### Step 5: 位置穩定性指數 [增強版]

**5.1 基本：** 審視近 10 仗中跑入前三名的比例：多於一半 = 鐵腳；大部分都入三甲 = 超級鐵腳
**5.2 比較排名：** 與同場對手比較穩定性 → `STABILITY_RANK`（全場最穩定 / 中段 / 最不穩定）；觀察近 6 仗名次起伏幅度判斷波動類型：穩定 / 一般 / 大起大落
**5.3 聯動：** Step 3 新鮮度風險 → 降低穩定性信心一級
**5.4 [新增] 場地專家辨識 (Track Specialist Detection)：** 若馬匹在「同場 + 同程」具備 ≥5 次勝出紀錄，標記為 `[場地專家 (Track Specialist)]`。即使近績級數下滑，此類馬匹在步速崩潰情境下往往憑經驗穩定走勢，不可完全忽略。詳見 Step 14.3 「場地專家保底」。
**5.5 [新增] 條件型馬辨識 (Condition Specialist Detection)：** 若馬匹在「同場 + 同程」具備 ≥2 次勝出紀錄，但非首本場地/路程的近績大起大落（如 7-1-9-6-4-1），標記為 `[條件型馬 (Condition Specialist)]`。此類馬並非真正不穩，而是**極度依賴特定情境**才能發揮。若今仗恰好符合其勝出條件（同場 + 同程 + 有利檔位），穩定性維度判斷強制從 ❌ 覆蓋為 ➖ Neutral（不再因非首本賽事的波動而受罰）。

### Step 6: 隱藏變數與速勢

**引用 Step 0：** 用 `PACE_TYPE` 判斷跑法受惠/受損，用 `BIAS_DIRECTION` 判斷逆偏差
- 配備/負磅變動（> 5 磅）
- 末段分析：慢步速但末段追近 = 馬匹實際能力強（被步速拖累的「隱藏強馬」）；快步速但仍能貫注到底 = 確認強馬
- 標記「追勢有餘 (Eye-catching)」

**場地狀態影響 (Going Impact)：**
- 若今日 Going = Good-to-Yielding 或 Yielding：馬匹若**無任何**濕地/偏軟場地出賽紀錄 → 標記 ⚠️「場地適性未知」，在評級矩陣「路程/新鮮度」維度額外考慮
- 若馬匹有同等或更軟場地下的贏馬/上名紀錄 → 場地適性已驗證，可作為正面參考
- 若馬匹在濕地/偏軟場地有包尾大敗紀錄 → 場地適性已驗證為不利，在評級矩陣中作為負面因素

**Going 與 AWT 互動邏輯 (Going/AWT Interaction) [新增]：**
- 若草地 Going 為 Yielding/Soft 但賽事在 AWT 泥地舉行 → Going **不影響** AWT 賽事（全天候跑道）
- 若馬匹草地濕地績差，但 AWT 績優 → 草地 Going 不影響 AWT 評估
- 嚴禁將草地 Going 判斷套用於 AWT 賽事，避免產生虛假❌

**距離轉換風險矩陣 (Distance Transition Risk) [新增]：**

| 距離變動 | 風險等級 | 處理 |
|:---|:---|:---|
| ±0-100m | 忽略 | 無需調整 |
| +200m | 低 | 檢查 Sire AWD + 近績表現 |
| +400m 或以上 | ⚠️ 高 | 標記 `[增程大躍進]`。豁免：曾在同等/更長距離績優；上仗輕鬆大勝；Sire 為 Stamina 型 |
| -200m 至 -400m | 中 | 檢查馬匹能否縮步幅，純均速型縮程 = 高風險 |
| 首次跑此距離 | 低至中 | 檢查是否已證明相近距離 (±100m) |

**[實績覆蓋血統 (Proven Class Override)]：**
- 若馬匹在某距離/場地已有 **≥2 次勝出** 或 **≥3 次上名**，其實際賽績**完全覆蓋** Sire AWD 的理論投射。即使 Sire AWD 暗示該馬不適合此距離/場地，實績已充分證明其能力，**嚴禁**以血統為由降級或質疑距離適性。
- Sire AWD 分析僅作為**首出馬/輕驗馬 (<5 場正式賽績) 或首次嘗試新距離/場地**的主要參考工具。
- 對於已有充分實績的馬匹，Sire AWD 降格為「背景資訊」，**不可**用作封頂評級或觸發距離風險警示。

**跑馬地專家矩陣 (Happy Valley Specialist) [增強]：**
- 跑馬地贏馬數據應獲 1.5× 權重（跑馬地為獨特小場，場地經驗價值極高）
- 跑馬地 1000m 起步即入彎，檔位 ≥8 = 強制加入「致命死檔」微調降級因素，除非已證明跑馬地 1000m 外檔能力
- 跑馬地夜賽因素：若馬匹有專屬夜賽贏馬紀錄 → 可作為輔助正面參考
- **[五班谷草特例 (Class 5 HV Bias)]：** 跑馬地第五班賽事，檔位 1-3 檔馬匹具備顯著地利（五班馬直路衝刺力弱，全靠沿途慳位）。此優勢是極強的情境加分，但不可無理越級（評級仍是最高標準，僅在同級評分中作優先排名考慮）。

### Step 8: 級數與部署加權

**8.1 級數軌跡：** 降班馬偵測 → 高班輸頭馬 < 4.0 馬位 → `「超級降班馬」` 評級升一級

**8.1B 級數天花板偵測 (Class Ceiling Detection) [新增]：**
- **評分見頂警示：** 若馬匹評分已達其班次上限 (如第三班評分 78-80) 且近績競爭力不足 → 標記 `[評分見頂 (Rating Ceiling)]`
- **班次壓縮：** 第四/五班評分差異極細微，此時負重與檔位重要性不成比例地放大（評分差異可忽略，勝負取決於戰術執行）
- **分級賽降級檢查：** 若馬匹由第二班降至第三班，區分「班次不配 (Class Mismatch)」與「真正狀態下滑 (Form Decline)」— 前者可給予降班優勢，後者不可

### Step 9: 負重與班次協同分析

> **注：班次與負重協同僅作微調因素（見 Step 14.2B），不直接改變核心評級的 ✅/❌，以免佔比過重。**

| 檢查 | 邏輯 |
|:---|:---|
| 負重變動 | 加磅 > 5lb → 檢查歷史同等負重表现 |
| 升班輕磅 | 升班但負磅驟降（如 135lb → 115lb） → 判斷輕磅優勢能否補償班次差距 |
| 降班死磅 | 降班但需負頂磅（133-135lb） → 重磅可能拖累降班優勢 |
| 減磅優勢 | 見習騎師紅利 + 前領型 → `Weight Drop Advantage` |
| 頂磅考驗 | 133-135lb 且非降班 → 級數能否克服死磅 |
| 五班輕磅紅利 | 第五班賽事中，負 120lb 或以下 且排 1-4 檔的馬匹，具備極強競爭力（輕磅在五班快步速中最佔優）。必須強制列入潛在機會名單。 |
| 二班負重連勝懲罰 | 若馬匹上仗贏馬後，今仗負磅 > 130lb 且初次面對該班次頂尖評分馬時（特別是 1800m 或以上），連勝難度極高。評級最高封頂 **B+**，並必須在「最大失敗風險」標註「評分飽和風險」。 |
