**根本原因：引擎缺乏「級別折算 (Class Discount)」機制**

引擎目前的評級矩陣以✅數量作為評級基礎，但**不區分**那些✅是在甚麼級別的賽事中取得的。Burj 的前兩仗連捷均來自 Canterbury 週三夜賽和 Wyong 省級日賽：

- Canterbury 1250m 對手級數 ≈ BM68
- Wyong 1350m 對手級數 ≈ BM72

而今場是 **BM78 都會賽**，相差至少 1-2 個班次。引擎直接將省級連勝的「動量指標 (Momentum Score)」等價於同級別的連勝，導致輸出了膨脹的 A+ 評級。

對比同場：
- **Formal Display**：82 分、BM78 原場戰將 → 被引擎降至 C+
- **Burj**：77 分、省級連勝馬 → 被引擎升至 A+
- 兩者評分差 **5 分**，但引擎排名完全倒轉

> [!WARNING]
> **SIP-AU16 需求觸發：** 此案例證明引擎需要一個「班次折算 (Class Discount Factor)」— 當馬匹的近績主要來自低於目前賽事 ≥1 個班次的賽事時，所有「狀態/段速/賽績線」的✅必須降一級為➖，以防止省級數據膨脹為都會賽頂級評級。

## 🚀 4. 策略改進方案（SIP-AU v5.2）

### SIP-AU09：歷史小樣本過度懲罰修正 (History Small-Sample Override)

**問題描述：**
引擎對「同程歷史賽績」賦予過高的降級權重，即使樣本量極小（n≤5）且馬匹具備壓倒性的班次優勢。

**觸發事件：**
- R8 Formal Display：「Second-up 3戰0名 → C+」，但 82 分評分全場最高
- R5 Fine Vintage：「1000m 3戰0勝 → C」，但淋地 10戰3勝且季內狀態穩定

**修正方案：**
```
IF (歷史負面樣本 n ≤ 5) AND (馬匹評分 ≥ 同場 Top 3):
    歷史懲罰權重 × 0.5 (減半)
    觸發 [CLASS_OVERRIDE] 標記
    基礎評級下限 = B-（不得低於此級）
```

**預期效果：** R8 的 Formal Display 將從 C+ 升至 B+，保留在 Top 4 候選池內。

### SIP-AU10：Second-up 週期動態評估 (Second-up Dynamic Assessment)

**問題描述：**
引擎將「Second-up 不勝」作為機械式核心❌懲罰，未考慮當下的場地適應力、班次差距等壓倒性正面因素。

**觸發事件：**
- R8 Formal Display：Soft 3戰2勝 + 全場最高評分 82 → 但因 Second-up (0/3) 被核心❌降級

**修正方案：**
```
IF (Second-up 歷史負面) BUT (以下任一條件成立):
    (a) 場地適應力 ≥ 2勝/3場@Soft
    (b) 評分差距 ≥ 同場平均 +4分
    (c) 具備 First-up 勝績（新鮮感馬特徵）
THEN:
    Second-up 懲罰降至 [半核心➖] 而非 [核心❌]
    輸出 [SECOND_UP_MITIGATED] 標記
```

---

### SIP-AU11：冠軍級騎練反彈防禦牆 (Elite Combo Rebound Shield)

**問題描述：**
引擎將 `Chris Waller + J-Mac` 等冠軍級騎練組合的「近績崩潰」等同於一般騎練，但冠軍組合的「賽前一晚決策」靈敏度及反彈率遠高於數據面所示。

**觸發事件：**
- R8 Emirate：Chris Waller + J-Mac → D 級（「市場陷阱」），實際跑亞軍

```
IF (騎師 ∈ {J-Mac, Tom Berry, K-McEvoy} AND 練馬師 ∈ {Waller, Waterhouse, Snowden, Maher}):
    評級下限 = B-（絕不允許降至 C/D）
    觸發 [ELITE_COMBO_FLOOR] 標記
    同時在分析結論中輸出：「⚠️ 冠軍組合下限保護：近績雖差但反彈率極高」
```

### SIP-AU12：外檔 × 長途 × 騎師交叉加成 (Wide Barrier × Distance × Jockey Synergy)

**問題描述：**
引擎對外檔（Barrier 7+）的懲罰在所有距離和騎師配搭下均相同，但在 1600m 以上長途賽事中，擅長留後且有硬功的騎師（如 Jason Collett、Tom Marquand）能有效抵消外檔劣勢。

**觸發事件：**
- R3 Mornington Pier：Jason Collett + 8檔 → B 級未入 Top 4，但最終贏馬
- R4 Cormac T：Mollie Fitzgerald 9檔 + 55.5kg → B- 未入選，但跑亞軍

**修正方案：**
```
IF (距離 ≥ 1600m) AND (騎師 ∈ {留後型騎師名單}):
    外檔懲罰(EEM❌) 降級至 (EEM➖)
    
IF (距離 ≥ 1600m) AND (減磅 ≥ 3kg):
    輕磅加成觸發 [LIGHTWEIGHT_LONG_ROUTE] ✅
    強制列入 Top 4 候選池邊緣位
```

### SIP-AU13：班次差距覆蓋規則 (Class Override Rule)

**問題描述：**
引擎缺乏「班次差距 = 物理優勢」的系統性覆蓋機制。當一匹馬的評分遠高於同場平均值時，其多數負面因素（如不利檔位、歷史劣績）應被顯著抵消。

**觸發事件：**
- R8 Formal Display：82 分 vs 同場平均 ≈75 分，差距 +7 分
- R5 Wondereach：79 分全場最高，引擎已給 A- 但在形勢中落敗

**修正方案：**
```
IF (馬匹評分 - 同場中位數 ≥ 5):
    所有 [輔助] 維度自動 ✅（班次壓制）
    基礎評級下限 = A-
    觸發 [CLASS_DOMINANCE] 標記
    
IF (馬匹評分 = 同場最高) AND (差距 ≥ 7):
```

### SIP-AU16：省級數據班次折算 (Class Discount Factor) 🆕

**問題描述：**
引擎的 8 維度評級矩陣（✅/❌/➖）不區分賽績來源的級別。當一匹馬的近績主要來自較低級別賽事（如省級 BM68/BM72），引擎會將其「連勝動量」、「段速質量」、「賽績線」等指標直接等價為都會 BM78+ 級別，導致評級嚴重膨脹。

**觸發事件：**
- R8 Burj：省級 Canterbury (BM68) + Wyong (BM72) 兩連勝 → 引擎授予 A+（全日最高）
- 同場 Formal Display 持有 82 分全場最高評分（BM78 原場戰將）→ 卻被降至 C+
- 最終 Burj 未能上名，Formal Display 贏馬

**修正方案：**
```
STEP 1: 計算「班次差距 (Class Delta)」
    Class_Delta = 今場賽事級別 - 近績主要級別
    例如：BM78 - BM72 = +6 (差距大)

STEP 2: 若 Class_Delta ≥ 4:
    以下維度的✅自動降級為➖：
    - 狀態與穩定性 [核心]
    - 段速與引擎 [核心]
    - 賽績線 [輔助]
    觸發 [CLASS_DISCOUNT] 標記

STEP 3: 評級上限封頂
    IF Class_Delta ≥ 4 AND 馬匹評分 < 同場 Top 3:
        評級上限 = A-（禁止授予 A/A+/S-/S）
```

**預期效果：**
- Burj 的 8✅ → 降至 5✅（3 個省級外推✅被降級）→ 基礎評級從 A+ 降至 B+/A-
- 同時 Formal Display 因 SIP-AU09/AU10 修正後升至 B+ → 兩者排名回歸合理
