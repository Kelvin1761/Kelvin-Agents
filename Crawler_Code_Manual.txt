------------------------------------------------------------
第一部分：網路爬蟲 (Web Crawler / Scraper) 的通用核心技術
------------------------------------------------------------
當我們在寫爬蟲程式時，一般的核心流程永遠逃不開 ETL (Extract, Transform, Load) 以及「對抗防禦機制」：

1. HTTP 請求與偽裝 (HTTP Request & Spoofing)
一般的 requests 套件很容易被現在的網站 (如 Cloudflare, Akamai) 阻擋，因為缺乏真實瀏覽器的指紋。
- 解決方案：現代爬蟲常必須修改 User-Agent、Header，甚至改變底層的 TLS (傳輸層安全協議) 特徵，才能讓伺服器以為這是一個真實的人類瀏覽器。

2. 內容渲染與狀態提取 (Rendering & State Extraction)
過去的爬蟲使用 BeautifulSoup 刮取 HTML 標籤。但現在的網站多為 React, Vue 等單頁應用 (SPA)，畫面都是靠 JavaScript 動態生成的。
- 解決方案：
  A. 直接攔截 API (RESTful 或 GraphQL) 傳輸的 JSON 檔案。
  B. 使用 無頭瀏覽器 (Headless Browser 如 Selenium, Playwright) 模擬真人在背後把網頁打開，等 JavaScript 跑完後再抽資料。
  C. 攔截 SSR (伺服器端渲染) 的初始狀態 (如 window.__NUXT__ 或 __NEXT_DATA__)，裡面通常塞滿了純淨的資料庫內容。

3. 資料清理與組裝 (Data Transformation)
從網頁或 JSON 抓下來的資料通常很雜亂，爬蟲必須進行清理：例如將字串 "12.5kg" 轉換為浮點數 12.5、將時間戳記換算成當下日期、組合不同 API 抓到的片段資料。

4. 輸出與儲存 (Storage / Load)
整理好的數據最後需要儲存，可能是寫入純文字檔 (.txt/.md)、存入 CSV / Excel Excel，或者寫入關聯式資料庫 (SQL / Firebase) 供儀表板使用。


------------------------------------------------------------
第二部分：你的 Au Race Extractor (extractor.py) 實戰拆解
------------------------------------------------------------
了解了通用原理後，你會發現這支程式幾乎用上了目前最頂級的現代爬蟲技術來繞過 Racenet 的防護機制：

💡 1. 繞過高級反機器人防護 (Anti-Bot Bypass)
- 它是如何做到的？
  程式使用了 curl_cffi，並設定 impersonate="chrome120"。這個套件在網路最底層模擬了 Google Chrome 第 120 版的 TLS 握手行為。因此 Cloudflare 完全無法察覺這是 Python 機器人，直接放行，成功抓下源代碼。

💡 2. 混合型渲染提取術 (Hybrid Playwright Extraction)
- 它是如何做到的？
  這支程式「沒有」讓 Playwright 直接對網路上網 (因為這速度慢且容易被抓)。它的做法極其聰明：
  A. 先把繞過防護抓到的源代碼，存成「本機端」的 HTML 檔案。
  B. 用 Playwright 打開本機的檔案，執行 JavaScript。
  C. 透過 JS 攔截網頁深處的 window.__NUXT__ 變數 (這是 Vue/Nuxt.js SSR 架構的快取狀態)。
  透過這種作法，我們秒殺了所有的反爬蟲，並得到了 100% 完整的原生 JSON 數據庫 (Apollo GraphQL Cache)！


------------------------------------------------------------
第三部分：你的爬蟲代碼核心函數逐個拆解
------------------------------------------------------------

【函數 1：fetch_nuxt_data (數據攔截器)】
> 職責：發起 HTTP 請求，偽裝身分，並利用 Playwright 榨取 JSON。
- 步驟 A：下載源代碼到 racenet_temp_[timestamp].html。
- 步驟 B：打開隱藏瀏覽器執行 JS，提取資料庫。
- 步驟 C：刪除本機暫存檔，無痕結束。

【函數 2：extract_event_metadata (賽事元數據提取器)】
> 職責：在龐大的 JSON 中，尋找單場賽事的頂層資訊。
- 自動遍歷資料，尋找 Event: 或 $Event: 節點。
- 提取出賽事層次的天氣狀況 (溫度)、場地掛牌 (Good 4, Soft 5)、獎金、距離等整體資訊。

【函數 3：process_race (個別賽局處理引擎)】
> 職責：這是整個爬蟲最核心的「資料轉換 (Transform)」地帶。
它會自動歷遍出賽名單 (selections)，抽絲剝繭：
1. 過濾與除錯：直接捨棄狀態為 Scratched 的退出馬匹。
2. 數學計算：把基礎負重 (Base Weight) 加上見習騎師的減磅 (Claim Weight) 自動算成「實際負磅」。
3. 關聯資料合併：一匹馬的資料散落在各處，程式會去抓練馬師、騎師過去一年的勝率，以及馬匹父系、母系的血統資料合併起來。
4. 過往賽績深度提取：包含以往跑過的場地狀態績效 (Good, Soft, Heavy)、歷史賽事的時間、勝負距離、影片簡評、甚至競賽委員會報告 (Stewards Report)。

【函數 4：process_meeting (全日賽事調度器)】
> 職責：控制全日所有場次的排程器。
1. 定位目標：先進入總覽頁 (Overview)，找出今天共有幾場 (Event IDs)。
2. 智慧過濾：允許你輸入終端機指令 (例如只要抓 race 1-4，就不會浪費時間全抓)。
3. 一鍵統整：統一呼叫前面的函數，最後在背景建立專屬日期馬場的資料夾，吐出 Racecard 與 Formguide 分析報告。

總結
------------------------------------------------------------
一般的爬蟲可能只會寫簡單的 BeautifulSoup 去抓 div 標籤，那非常容易因為對方網站改版而壞掉。
而你所使用的這支 Crawler，結合了「底層網路偽裝」以及「前端框架狀態竊取 (NUXT State)」，直接繞過了視覺層，從資料庫層級提取最純淨的 GraphQL 數據，是非常頂級且穩定的現代爬蟲架構典範。
