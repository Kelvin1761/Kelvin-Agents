🧠 三、Python Math Engine (核心突破)
============================================================

以前所有數學運算（命中率、CoV、+EV）靠 LLM 自己估...
LLM 做數學出咗名唔準，計出嚟嘅數經常錯！
而家 100% 由 Python 精確計算，LLM 淨係讀成品數字就得。

由 Python 負責嘅計算：
  ✅ AVG / MED / SD / CoV — 基礎統計量
  ✅ L5 / L3 均值 — 近期走勢
  ✅ 穩膽線 (Banker Line) — AVG - 0.5*SD
  ✅ 價值線 (Value Line) — MED + 0.5*SD
  ✅ Bet365 盤口階梯命中率 — 針對 PTS/REB/AST/STL/BLK/FG3M
     每條線嘅 L10/L5/L3 命中率全部由 Python 逐場點算
  ✅ AMC (Average Margin of Clearing) — 過線平均超出幅度
  ✅ +EV Edge — (預估勝率 - 隱含勝率)
  ✅ Pace-Adjusted Projection — (對手PACE-聯盟均PACE)/均PACE * AVG
  ✅ Under 自動偵測 — 當 Over 命中率 ≤40% 時自動生成 Under 候選
  ✅ CoV 分級標籤 — 🛡️極度穩定 / ✅穩定 / ➖一般波動 / 🎲神經刀

示範 (Devin Booker PTS)：
  L10: [34, 36, 26, 22, 25, 14, 22, 34, 40, 31]
  AVG: 28.4 | MED: 28.5 | SD: 7.54 | CoV: 0.265 (➖一般波動)
  Banker Line: 24.6 | Value Line: 32.3
  Over 24.5 → L10=70% L5=80% L3=100% | AMC=7.8 | Edge=+17.4%
  Under 34.5 → L10=80% | 🛡️ 穩膽 Under

📈 四、模塊 D 進階分析升級
============================================================

D1. Usage Redistribution (傷缺紅利計算)
    ─ 當核心球員傷咗唔打嗰陣，Python 自動計算佢嘅 USG%
      點樣按比例重新分配俾其他在場球員
    ─ 例如：如果 Kevin Durant (USG 30%) 傷缺，
      Booker 嘅預計 USG 會自動升高

D2. Rest Day Fatigue Model (疲勞度模型)
    ─ 從 nba_api 提取球員喺 0/1/2/3+ Days Rest 嘅場均差異
    ─ 如果 B2B (0 Days Rest) 嘅 PPG 顯著下降 (>10%)，
      自動生成 ⚠️ B2B 疲勞警告

D3. Correlation Warning System (相關性警告)
    ─ 自動偵測 SGM 組合入面嘅邏輯衝突：
      🚨 Cannibal Risk: 同隊 3+ 高球權球員全買 PTS Over
      ✅ 正相關: 控衛 AST Over + 射手 PTS Over
      🚨 矛盾: 球員 PTS Over + 同隊 Total Under

D4. H2H 歷史對戰數據 (全新加入)
    ─ 從 nba_api 提取球員近 2 季對住特定對手嘅完整 Box Score
    ─ Python 自動計算 H2H 均值、高低範圍
    ─ 同 L10 均值比較，判斷「宰殺型」定「被壓制型」
    ─ 示範 (Booker vs CHA):
      近 3 場: 30, 30, 39 → H2H AVG: 33.0 (遠高過 L10 AVG 28.4!)
      = 📈 H2H 加成 → 信心分 +1
