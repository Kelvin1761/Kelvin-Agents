## 先講結論
今次 3 個 meeting 最核心嘅問題，唔係單純「要加 jockey_trainer / class_distance / draw_pace / sectional」咁簡單。

真正問題係：

1. 模型有一批馬 **高估咗**
2. 亦有一批馬 **低估咗**
3. 而高估同低估，背後好多時係同一組訊號結構失衡造成

即係話，問題唔係某一個 factor 完全冇用，而係：
- 有啲 factor 會將一啲馬推得太前
- 同一時間，另一啲真正應該入前列嘅馬，因為某幾項分數太低，被壓咗落去

## 三場整體形態
- Caulfield：`Gold 1 / Good 1 / Pass 1 / 1 Hit 2 / Miss 4`
- Eagle Farm：`Good 1 / Pass 2 / Miss 6`
- Rosehill Gardens：`Good 1 / Pass 1 / 1 Hit 5 / Miss 3`

整體判斷：
- `20` 場屬於 `clean model failure`
- `8` 場屬於 `帶有可寬恕元素或非純模型錯誤`

即係大多數失手都真係模型排序問題，唔係單靠賽事事故可以解釋。

## 高估定低估？
我用咗 3 個 meeting 嘅 `Meeting_Auto_Scoring.csv` 同實際賽果對過：

### 低估實際入前三馬（underrated）
- Caulfield：`17` 隻
  - 其中 `10` 隻其實只係排第 `4-6`
  - `7` 隻先係真係跌到第 `7+`
- Eagle Farm：`20` 隻
  - 只有 `4` 隻排第 `4-6`
  - `16` 隻跌到第 `7+`
- Rosehill Gardens：`21` 隻
  - `11` 隻排第 `4-6`
  - `10` 隻跌到第 `7+`

解讀：
- Caulfield 同 Rosehill 主要係 **有訊號但排序未推上去**
- Eagle Farm 就唔止係 tie-break 問題，係 **真係有一批馬被壓得太後**

### 高估模型 Top 3 馬（overrated）
- 3 個 meeting 合共都有大量模型 Top 3 最終未入實際前三
- `19` 場係連模型第一選都未能上名

解讀：
- 問題唔止係「漏捉咗黑馬」
- 仲包括「將本來唔應該排咁前嘅馬，推得太前」

## 四個主題其實代表咩問題？

### 1. `jockey_trainer`
呢個主題出現最多，但唔代表「只要加 jockey / trainer 分就得」。

更準確嘅講法係：
- 模型現時有啲馬，會被 `confidence_score`、`consistency_score` 呢類較軟性訊號推上去
- 但真正反映部署、配搭、出擊意圖嗰類訊號，未足夠將應該上位嘅馬推前

換句話講：
- **被低估嘅馬**：可能係真有部署 / 人馬配搭優勢，但現有欄位未夠強或者未夠細緻去反映
- **被高估嘅馬**：可能只係 profile 靚、近況穩、信心分高，但實際部署強度未到咁前

所以 `jockey_trainer` 背後唔係純粹「少咗數據」，而係：
- 現有 proxy 未夠 sharp
- 軟性 profile signal 太容易蓋過真正部署 signal

### 2. `class_distance`
呢個主題高頻出現，特別喺 Rosehill 同 Eagle Farm。

佢反映嘅唔一定係「班次完全睇錯」，而係：
- 模型可能高估咗某啲表面 form line 漂亮、但實際對今場班次 / 路程唔係咁啱嘅馬
- 同時低估咗一啲轉返合適路程、或者 previous form line 強過表面數字顯示嘅馬

所以：
- **高估**：舊 form line、穩定性、信心分幫咗手，但今場 class / trip context 未必真係配合
- **低估**：真實 race strength、路程適配、班次含金量未被充分反映

更關鍵係，archive backtest 已經話畀我哋知：
- 一個太 broad 嘅 `class_venue_weight_soft` 版本會明顯拖低整體成績

即係代表：
- 問題係真
- 但唔可以用粗暴全局加權去解

### 3. `draw_pace`
呢個係 Caulfield 特別明顯，Rosehill 都唔少。

我用 factor 分佈睇，發現一個幾重要嘅現象：
- 被低估馬最常見低分項之一：`pace_map_score`
- 被高估馬最常見低分項之一：都係 `pace_map_score`

呢個意思唔係 pace 無用，而係現時 pace signal 可能有兩個問題：

1. 有時模型明知某匹馬 pace 唔理想，但其他軟性分數照樣將佢頂上前列
2. 另一邊，真正應該因 pace / run-style / barrier context 受惠嘅馬，又未被夠力推上去

即係現況比較似：
- `pace_map_score` 有訊號
- 但佢未成為一個穩定、夠決定性嘅排序約束條件

所以 `draw_pace` 問題，本質上係：
- 唔係完全無 data
- 而係現有 pace / draw context 對最終排序嘅 influence 未夠一致

### 4. `sectional`
`sectional` 排第 4，但唔代表佢唔重要。

我見到：
- 被低估馬最常見低分項入面，`sectional_score` 非常高頻
- 被高估馬最常見低分項入面，`sectional_score` 一樣高頻

呢個代表咩？

代表問題唔係「純粹加強 sectional 就得」。

更可能係：
- sectional coverage / interpretation 本身有 noise
- 有啲馬因 sectional 差被壓低，但實際其他 context 應該補返
- 有啲馬雖然 sectional 一般，但被其他 soft factors 推得太前

所以 `sectional` 喺呢 3 場入面，比較似：
- 一個重要但唔夠穩定嘅校正信號
- 未適合單獨做第一優先 SIP

## 用數據睇，現時模型最似點樣錯？
如果睇被低估／高估馬最常見分項模式：

### 被低估馬最常見低分項
- `form_score`
- `pace_map_score`
- `sectional_score`
- `distance_score`

### 被低估馬最常見高分項
- `confidence_score`
- `consistency_score`
- `trial_score`

### 被高估馬最常見低分項
- `pace_map_score`
- `form_score`
- `sectional_score`
- `rating_score`

### 被高估馬最常見高分項
- `confidence_score`
- `consistency_score`
- `sectional_score`
- `formline_score`

最值得留意嘅地方：

1. `confidence_score` 同 `consistency_score` 喺高估同低估兩邊都非常高頻
2. `form_score`、`pace_map_score`、`sectional_score` 喺高估同低估兩邊都經常成為短板

解讀：
- 模型現時好可能 **過度信任 profile 型訊號**
  - 例如信心、穩定性、部分 formline 包裝
- 但對真正決定今場能唔能夠跑入前三嘅即場排序 signal
  - 例如步速形勢、實戰段速、今場路程適配
  未夠有力

所以實際問題唔係「某個 factor 冇」，
而係：
- **軟訊號太容易將馬推上去**
- **硬排序訊號未夠強去拉返啱**

## 我哋有冇 data 支持未來做得更準？
答案係：**有一部分有，亦有一部分明顯未夠。**

### 目前已經有嘅
- 每匹馬嘅 factor-level score
  - 例如 `form_score`、`pace_map_score`、`sectional_score`、`trainer_score` 等
- 每場完整 ranking 同 rank gap
- 實際賽果
- 部分 results JSON 包括：
  - `starting_price`
  - `running_positions`
  - `comments`
- archive-level backtest framework
  - 可以將任何新 shadow variant 放入 `47 meetings / 417 races` 驗證

呢啲已經足夠支持：
- 做高估 / 低估 bucket 分析
- 做 targeted shadow test
- 驗證某個 SIP 係咪真係改善，而唔係只 fit 單日 meeting

### 目前明顯唔夠嘅
- AU 官方 stewards / incident notes coverage 唔穩定
  - 所以我哋未能好穩定咁分辨：
    - 真模型失誤
    - 定係臨場事故 / 受阻 / 慢閘 / 外疊消耗
- `jockey_trainer` 目前更似 proxy，未必有足夠深度標示真正出擊 intent
- `class_distance` 有資料，但對班次含金量、form line 強弱、路程轉換難度嘅 normalization 未夠細
- `draw_pace` 有 score，但未有標準化「預期步速 vs 實際走位」驗證層
- `sectional` 有 signal，但 coverage 同解讀穩定性未必足夠做強主導

## 真正下一層結論
如果而家問：

### 問題係高估定低估？
答案係：
- 兩樣都有
- 但 Eagle Farm 偏向嚴重低估
- Caulfield / Rosehill 偏向「有訊號但未推前」加埋部分高估

### 高估 / 低估真正成因係咩？
答案係：
- 軟性 profile 訊號太易推馬上去
- 即場排序訊號未夠穩定咁拉返啱
- `jockey_trainer / class_distance / draw_pace / sectional` 其實唔係四個獨立壞點
- 佢哋更似四個角度，一齊指出同一個結構問題：
  - **排序層對 hard race context 反應未夠強**

### 我哋有冇 data 支持未來改善？
答案係：
- **有足夠 data 去做 targeted shadow validation**
- **未有足夠 data 去直接好有把握地下全局結論**

即係：
- 可以做實驗
- 可以做回測
- 可以分 bucket 驗證
- 但唔應該只靠呢 3 場就直接批 global mainline 改動
